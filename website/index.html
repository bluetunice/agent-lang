核心特性

AFL 涵盖了 Agent 开发所需的全部核心能力

🧠

LLM 原生集成

内置 LLM 调用，支持多轮对话、提示模板、流式响应、结构化输出。兼容 OpenAI、Anthropic 等主流模型。

🔍

知识库（RAG）

原生向量数据库集成，支持文档添加、语义检索、结果直接注入 LLM 上下文，快速构建 RAG 应用。

🔗

MCP 协议

支持 Model Context Protocol，标准化 Agent 工具调用。连接 MCP 服务器、调用工具、管理上下文。

🧩

Skill 技能系统

内置 math、text、data 三大预设技能，支持 .agent/.afl 文件导入和动态调用。复用 Agent 能力单元，开箱即用。

⚙️

系统命令

安全的系统命令执行，支持超时控制、管道操作、平台信息查询、命令存在性检查。

⚡

异步并发

async/await 异步语法，parallel 并行执行块，wait all/any 同步原语，充分利用并发能力。

📦

模块系统

支持代码导入导出，模块化组织。import/from/export 语法，条件导入，内置标准库。

🔄

流程控制

if/else if/else 条件分支，for/while 循环，break/continue，三元表达式，?? 空值合并。

🔧

内置测试框架

内置 test/suite 测试框架，assert 断言，assert_eq 值断言，test.run() 一键运行所有测试。

🌐

HTTP API

内置 HTTP 客户端，支持 GET/POST/PUT/DELETE，JSON 自动编解码，请求/响应拦截。

为什么选择 AFL？

传统方式
# Python + LangChain
from langchain.llms import OpenAI
from langchain.vectorstores import Chroma

llm = OpenAI(model="gpt-4")
db = Chroma(persist_directory="./kb")
docs = db.similarity_search(question)
context = "\n".join([d.page_content for d in docs])
prompt = f"Context: {context}\nQ: {question}"
answer = llm.predict(prompt)
print(answer)
AFL 方式
kb.connect("local", { path: "./kb" })
let docs = kb.search(question, top_k=3)

llm.new({ provider: "openai", model: "gpt-4" })
let answer = llm.prompt("Context: {{ctx}}\nQ: {{q}}", { ctx: docs, q: question })
print(answer)
